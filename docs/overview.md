# 无障碍图表工具概览

本文档用于快速了解项目提供的能力与模块划分，帮助你在开发或部署前掌握系统的整体形态。

## 核心模块

1. **用户管理**
   - 账号注册需填写姓名、邮箱和账号名，后台对邮箱与账号名做唯一性校验。
   - 登录支持使用邮箱或账号名两种方式，成功后返回 JWT 访问令牌。
   - 已登录用户可以在仪表盘内修改密码并立即生效。
2. **分组体系**
   - 分组以树形结构呈现，可创建任意层级的子分组。
   - 支持重命名、删除空分组以及通过树节点过滤任务列表。
3. **任务工作流**
   - 上传图表图片时指定所属分组与目标语言（Java 或 Kotlin）。
   - 仪表盘分页展示历史任务，并支持修改任务名称、调整分组或取消运行中的任务。
   - 任务详情页展示摘要、结构化数据和两种语言的代码片段，可保存自定义代码并下载压缩包。
4. **代码生成**
   - 后台线程模拟云端分析流程，生成摘要、数据点、表格数据以及 Java/Kotlin 代码。
   - 同步生成集成步骤文档，并在用户请求时封装为 zip 下载包。

## 技术栈

- 后端：Flask、Flask-JWT-Extended、SQLAlchemy
- 前端：Vue 3、Vite、Pinia、Axios
- 任务处理：内置线程队列模拟云端处理
- 数据库存储：默认 SQLite，可通过环境变量切换到 MySQL

## 启动顺序

1. 启动 Flask 后端（`flask --app app:create_app run`），自动初始化数据库表并启动后台线程。
2. 启动前端（`npm run dev`），通过代理访问后端 `/api` 接口。
3. 访问前端 `http://localhost:5173`（或控制台提示的端口），体验完整工作流。

## 目录结构

- `backend/`：Flask 应用、模型、蓝图、后台线程与工具函数。
- `frontend/`：Vue 3 单页应用及组件、状态、路由配置。
- `docs/`：项目说明文档与实现细节。

## 下一步可以做什么？

- 若要对接真实的图表分析服务，替换 `backend/utils/chart_processing.py` 中的模拟逻辑即可。
- 如需扩展更多语言或导出格式，可在任务模型中新增字段，并在详情页增加相应的标签页。
- 如果要部署到生产环境，推荐接入数据库迁移工具（如 Alembic）以及对象存储服务以替换本地文件系统。
